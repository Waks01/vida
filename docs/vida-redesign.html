07 · What changed

From the v1 spec to this one.

A redesign isn't a restyle. Here are the deliberate moves.

Vida design v1 — what it was

  • Flat documentationSix pages of horizontal phone rows. The screens sat on a single canvas like museum exhibits.
  • Numbered sections01 / 02 / 03 markers for every page. Decoration that didn't encode anything about the order.
  • One color palette per themeCSS variables, but a single flat surface tone per theme. Dark was one solid color.
  • Single Daily Check-in bannerOne commitment on the home screen — easy to ignore.
  • One horizontal shelf"Trending Now" was the whole catalog. Return users had nothing new to do.
  • 4 genre chipsA token gesture. Real short-drama apps show 8–12.
  • Coin cost on cards was missingReelShort hides the price; we matched them. The user discovers the cost by tapping.
  • Sections as a tab navClick Brand → Components → Themes → Screens. A document, not a product.

Cinematic v2 — what it is now

  • Vertical scroll, not a tab navThe page is the product walk. You scroll through it like a user would.
  • Named bands, no numbers"01 · The home screen", "02 · First launch", "03 · Returning users". The numbers are part of the narrative, not decoration.
  • Two-tone palette per themeInk (background) + Velvet (surface) per theme. A real sense of depth.
  • 2×2 Watch & Earn gridFour small commitments — check-in, ad, refer, spin — beat one big one. Same real estate, more taps.
  • Six shelves + Continue WatchingHero → Resume → Trending → Top 10 → Earn → Tropes. Six distinct next-actions in one scroll.
  • 12 trope chipsThe real ReelShort number, with hot/new color hints so the catalog feels alive.
  • Coin cost visible on every cardBottom-right of every poster. Users learn the price economy before they tap.
  • Mono as the signatureEvery number — coin balances, episode counts, view counts, rank digits — is JetBrains Mono. The user reads numbers as numbers.
